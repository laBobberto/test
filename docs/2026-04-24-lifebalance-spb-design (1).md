# LifeBalance SPb — Персональный городской ассистент

## Общая концепция

Веб-приложение (с возможной интеграцией в VK/MAX), которое помогает пользователям балансировать разные аспекты жизни в Петербурге через настраиваемые приоритеты, AI-планирование и геймификацию.

**Ключевая идея:** Пользователь сам определяет, что для него важно сейчас (учеба, карьера, здоровье, досуг, социализация), и система адаптируется под эти приоритеты, предлагая оптимальные активности и маршруты.

**Целевые аудитории:**
- Студенты (фокус на учебе и карьерном развитии)
- Жители (фокус на быте и досуге)
- Туристы (фокус на культуре и достопримечательностях)

**Основные функции:**
- AI-персонализация и рекомендации
- Умная навигация и маршруты
- Социальное взаимодействие через геймификацию

## Архитектура и роли

### Ролевая модель

Пользователь может выбрать одну или несколько ролей:

- **Студент**: фокус на учебе, карьерном развитии, студенческой жизни
- **Житель**: фокус на быте, досуге, городских сервисах
- **Турист**: фокус на достопримечательностях, культуре, маршрутах

Роли влияют на доступные приоритеты и типы рекомендаций.

### Система приоритетов

Пользователь распределяет важность между категориями (слайдеры или проценты):

- 📚 **Образование** (пары, самоподготовка, доп. курсы)
- 💼 **Карьера** (стажировки, вакансии, нетворкинг)
- 💪 **Здоровье** (спорт, сон, активность, питание)
- 🎭 **Досуг** (события, хобби, культура)
- 👥 **Социализация** (встречи, сообщества, мероприятия)
- 🏠 **Быт** (ЖКХ, покупки, бытовые дела)

AI учитывает эти приоритеты при планировании дня/недели.

## Основной функционал

### 1. Умное планирование с AI

AI-ассистент анализирует:
- Приоритеты пользователя
- Расписание занятий (для студентов)
- Данные трекера здоровья (сон, активность, пульс)
- События в городе
- Погоду и транспорт
- Историю активностей

И предлагает:
- Оптимальный план дня/недели
- Подходящие события и активности
- Маршруты с учетом всех факторов
- Предупреждения о перегрузке/недостатке активности

**Пример:** "Сегодня у тебя 3 пары, но трекер показывает плохой сон. Рекомендую легкую прогулку в обед (маршрут до Таврического сада 15 мин), вечером — спокойное мероприятие в библиотеке вместо активного события."

### 2. Интеграция данных

**Источники данных:**
- **Расписание ЛЭТИ**: автоматический импорт пар, учет свободного времени (API: digital.etu.ru/api/docs-public/)
- **Вакансии**: рекомендации стажировок/работы на основе изучаемых дисциплин и навыков (trudvsem.ru datasets)
- **События города**: фильтрация по интересам и приоритетам (researchinspb.ru/api/v1/public/event/)
- **Транспорт**: оптимальные маршруты между точками (API petersburg.ru)
- **Трекеры здоровья**: API интеграция (Apple Health, Google Fit, Xiaomi Mi Fit)
- **StreamTelecom данные**: анализ популярных мест и времени активности для рекомендаций (CSV: clicks, regions, campaigns)

### 3. Геймификация

**Механики:**
- **Баллы за баланс**: чем лучше соблюдаешь свои приоритеты, тем больше очков
- **Челленджи**: "Пройди 10к шагов + посети 2 новых места", "Неделя без пропусков пар"
- **Достижения**: бейджи за разные активности (культурный гуру, спортсмен, карьерист)
- **Стрики**: серии дней с выполненными целями
- **Рейтинги**: сравнение с друзьями (опционально)
- **Награды**: скидки от партнеров, доступ к закрытым событиям

**Расчет баллов:**
- Выполнение запланированных активностей: +10-50 баллов
- Соблюдение баланса приоритетов: +20 баллов/день
- Достижение целей по здоровью: +15 баллов
- Посещение новых мест: +25 баллов
- Стрики: множитель x1.5 после 7 дней
- Челленджи: 100-500 баллов

## Компоненты и технологии

### Frontend (Web-приложение)

- React/Vue.js с адаптивным дизайном
- Интерактивная карта (OpenStreetMap или Leaflet)
- Дашборд с приоритетами и статистикой
- Чат с AI-ассистентом
- Календарь и планировщик
- Профиль с достижениями

### Backend (локальный сервер)

- REST API
- База данных (PostgreSQL/SQLite)
- Кеширование (опционально)

### AI и интеграции

- Свои AI-провайдеры (OpenAI API, Anthropic, локальные модели)
- API расписания ЛЭТИ
- API "Я здесь живу" / petersburg.ru
- API трекеров здоровья (OAuth)
- Парсинг вакансий и событий

### Архитектура системы

```
Frontend (Web) <-> REST API <-> Backend Services
                                    ├─ AI Service (LLM провайдер)
                                    ├─ Data Aggregator (внешние API)
                                    ├─ Recommendation Engine
                                    ├─ Gamification Service
                                    └─ Database
```

### База данных (основные таблицы)

- `users`: профили, роли, настройки
- `priorities`: текущие приоритеты пользователя
- `activities`: история активностей
- `events`: события города (кеш)
- `schedules`: расписание занятий (кеш)
- `health_data`: данные трекеров
- `achievements`: достижения и баллы
- `recommendations`: история рекомендаций AI

### API endpoints (примеры)

- `POST /api/auth/register` - регистрация
- `GET /api/user/profile` - профиль пользователя
- `PUT /api/user/priorities` - обновление приоритетов
- `GET /api/plan/daily` - план на день
- `POST /api/plan/adjust` - корректировка плана
- `POST /api/chat` - чат с AI
- `GET /api/events` - события с фильтрами
- `GET /api/achievements` - достижения и баллы
- `POST /api/activity/complete` - отметка выполнения

### Интеграции (периодическое обновление)

- Расписание ЛЭТИ: обновление раз в день
- События: обновление каждые 6 часов
- Вакансии: обновление раз в неделю
- Транспорт: real-time или кеш на 15 минут
- Здоровье: синхронизация по запросу пользователя

## Пользовательский опыт

### Онбординг

1. Регистрация/вход
2. Выбор роли (студент/житель/турист или комбинация)
3. Настройка приоритетов (слайдеры по категориям)
4. Подключение интеграций:
   - Импорт расписания ЛЭТИ (номер группы)
   - Подключение трекера здоровья (опционально)
   - Выбор интересов и предпочтений

### Ежедневное использование

**Утром:**
- AI предлагает план дня на основе приоритетов, расписания, здоровья

**В течение дня:**
- Отметка выполненных активностей
- Получение уведомлений о событиях/возможностях
- Корректировка плана через чат с AI

**Вечером:**
- Подведение итогов
- Начисление баллов
- Обновление стриков

### Основные экраны

- **Главная**: план дня, текущие задачи, быстрые действия
- **Карта**: события, места, маршруты с фильтрами по приоритетам
- **Профиль**: приоритеты, статистика, достижения, настройки
- **Чат с AI**: вопросы, корректировка планов, рекомендации
- **Календарь**: недельный/месячный вид с активностями
- **Сообщество**: рейтинги, челленджи, друзья (опционально)

## AI-логика и алгоритмы

### Система рекомендаций

AI-ассистент принимает решения на основе:
- **Приоритеты пользователя** (веса категорий)
- **Контекст времени** (время суток, день недели, сезон)
- **Данные здоровья** (уровень энергии, качество сна, активность)
- **Расписание** (занятые/свободные слоты)
- **История** (что нравилось, что выполнялось)
- **Внешние факторы** (погода, загруженность мест)

### Алгоритм планирования

1. Загрузка фиксированных событий (пары, встречи)
2. Анализ свободных слотов
3. Оценка текущего состояния (здоровье, баланс приоритетов)
4. Подбор активностей под каждый слот с учетом:
   - Соответствия приоритетам
   - Физической возможности (расстояние, время)
   - Уровня энергии
   - Разнообразия
5. Оптимизация маршрутов между точками
6. Генерация плана с объяснениями

## MVP для хакатона

### Обязательный функционал (24-26 апреля)

1. Регистрация и выбор роли
2. Настройка приоритетов (слайдеры)
3. Импорт расписания ЛЭТИ (по номеру группы)
4. AI-генерация плана дня (через чат)
5. Отображение событий на карте
6. Базовая геймификация (баллы за активности)
7. Простой дашборд с статистикой

### Опциональный функционал (если успеем)

- Интеграция трекеров здоровья
- Рекомендации вакансий
- Достижения и стрики
- Оптимизация маршрутов
- Интеграция VK/MAX

### Тестовые данные

- 2-3 тестовых пользователя с разными ролями
- Реальное расписание группы ЛЭТИ
- События из API за текущую неделю
- Мок-данные трекера здоровья

### Демонстрация на защите (4 минуты)

1. Показать онбординг (30 сек)
2. Настройка приоритетов (30 сек)
3. AI генерирует план дня (1 мин)
4. Карта с событиями и маршрутом (1 мин)
5. Выполнение активности + начисление баллов (30 сек)
6. Итоговая статистика (30 сек)
