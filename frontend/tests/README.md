# Playwright E2E Testing Suite

## 📊 Обзор тестов

Полный набор E2E тестов для LifeBalance SPb, покрывающий все основные функции и взаимодействия пользователя.

## 🧪 Тестовые файлы

### 1. `e2e.spec.ts` - Основные флоу (30+ тестов)
- **Authentication Flow** - Вход и регистрация
- **Registration and Onboarding** - Полный флоу регистрации
- **Priorities Configuration** - Настройка приоритетов
- **Dashboard** - Главная страница
- **AI Chat** - Чат с ассистентом
- **Map Page** - Карта событий
- **Profile Page** - Профиль пользователя
- **Navigation** - Навигация между страницами
- **Responsive Design** - Адаптивность

### 2. `extended.spec.ts` - Расширенные тесты (40+ тестов)
- **Extended UI Interactions** - Продвинутые UI взаимодействия
- **Priorities Page Interactions** - Детальные тесты слайдеров
- **Dashboard Interactions** - Взаимодействия на дашборде
- **Chat Page Interactions** - Детальные тесты чата
- **Map Page Interactions** - Взаимодействия с картой
- **Profile Page Interactions** - Детальные тесты профиля
- **Error Handling** - Обработка ошибок
- **Accessibility** - Доступность
- **Performance** - Производительность

### 3. `advanced.spec.ts` - Продвинутые тесты (50+ тестов)
- **Advanced Form Interactions** - Копирование, вставка, автозаполнение
- **Onboarding Edge Cases** - Граничные случаи онбординга
- **Priorities Advanced** - Продвинутые тесты приоритетов
- **Dashboard Advanced** - Продвинутые функции дашборда
- **Chat Advanced** - Многострочные сообщения, спецсимволы
- **Map Advanced** - Продвинутые взаимодействия с картой
- **Profile Advanced** - Продвинутые функции профиля
- **Cross-page State** - Сохранение состояния между страницами
- **Animations** - Анимации и переходы
- **Data Loading States** - Состояния загрузки данных

## 🚀 Запуск тестов

```bash

```

### С UI интерфейсом
```bash
npm run test:ui
```

### В headed режиме (видимый браузер)
```bash
npm run test:headed
```

### Конкретный файл
```bash
npx playwright test tests/e2e.spec.ts
```

### Конкретный тест
```bash
npx playwright test -g "should display login page"
```

### Только один браузер
```bash
npx playwright test --project=chromium
```

## 📝 Best Practices

1. **Используйте data-testid** для стабильных селекторов
2. **Ждите элементы** вместо фиксированных таймаутов
3. **Изолируйте тесты** - каждый тест независим
4. **Очищайте состояние** после каждого теста
5. **Используйте Page Object Model** для сложных страниц
6. **Проверяйте accessibility** в каждом тесте
7. **Тестируйте на разных браузерах**
8. **Делайте скриншоты** при ошибках

### GitHub Actions пример
```yaml

  run: npm ci
  
- name: Install Playwright
  run: npx playwright install --with-deps
  
- name: Run tests
  run: npm run test
  
- name: Upload report
  uses: actions/upload-artifact@v3
  with:
    name: playwright-report
    path: playwright-report/
```
